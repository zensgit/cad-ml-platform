## 2. 蒸馏 v5b 超参改进

### 2.1 v5 失败原因分析

| 参数 | v5 值 | 问题 |
|------|-------|------|
| alpha | 0.3 | hard label 权重太低，学生丢失类别边界 |
| temperature | 3.0 | soft labels 过于平滑，类间区分度丢失 |
| lr | 5e-4 | warm start 下学习率过高，v4 特征被快速覆盖 |

### 2.2 v5b 改进配置

| 参数 | v5 | v5b | 改进原因 |
|------|-----|------|---------|
| alpha | 0.3 | **0.7** | 70% hard label 保持类别边界 |
| temperature | 3.0 | **1.5** | 保留更多类间区分度 |
| lr | 5e-4 | **1e-4** | 更保守，保持 v4 预训练特征 |

```bash
python scripts/distill_ensemble.py \
    --temperature 1.5 --alpha 0.7 --lr 1e-4 \
    --output models/graph2d_distilled_v5b.pth
```

### 2.3 预期

- v5（alpha=0.3, T=3.0）：91.0%
- v5b（alpha=0.7, T=1.5）：**预期 92-93%**
- 目标：单模型接近集成精度，但推理仅需 ~1ms（INT8）

### 3.2 关键新特征说明

**连通分量**（BFS 实现）：
```python
# 连通分量数：法兰（1个大连通体）vs 装配图（多个分离件）

**Hub 比率**：度数 > mean+2σ 的节点比例（轴类/法兰的中心节点模式不同）

**双向边比率**：无向边在 DXF 图中的比例（不同图生成策略产生的拓扑不同）

## 5. 蒸馏 v5b 最终结果

```

Epoch 11: train=0.920  val=0.906 ✓  ← best

### v5b vs v5 vs v4 对比

| 版本 | alpha | T | lr | Val Acc |
|------|-------|---|-----|---------|
| v4（无蒸馏） | — | — | — | **91.9%** |
| v5 | 0.3 | 3.0 | 5e-4 | 91.0% |
| **v5b** | **0.7** | **1.5** | **1e-4** | **90.6%** |

### 蒸馏失败根因

**v4 已到达该数据分布的训练上限**。从已充分训练的 checkpoint warm-start 蒸馏，soft label 信号反而干扰了已优化的决策边界。蒸馏更适合从 scratch 训练（让学生自由学习 soft 分布），而非覆盖已有的强特征。

### 最终结论

**蒸馏路线在当前条件下不可行。异构集成（95.8%）是确认的最优方案。**

若需单模型部署，v4 INT8（91.9%, 430KB）仍是最佳选择。
