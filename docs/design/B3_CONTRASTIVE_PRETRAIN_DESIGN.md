# B3: Graph2D Contrastive Pretraining Design

## Problem: Extreme Data Scarcity

- 110 labeled DXF files across 47 classes = avg 2.3 samples per class
- Graph2D supervised accuracy: ~27% (essentially random for 47 classes)
- Acquiring more labeled data is expensive (expert annotation of CAD drawings)
- BUT: many unlabeled DXF files may exist on disk

## Solution: Self-Supervised Contrastive Pretraining (GraphCL)

Contrastive pretraining learns useful graph representations from **unlabeled** data.
The core idea: two augmented views of the same graph should have similar embeddings,
while views of different graphs should be dissimilar.

References:
- You et al., "Graph Contrastive Learning with Augmentations" (NeurIPS 2020)
  - Reports 5-10% accuracy improvement on molecular/social graph benchmarks
  - Best gains in low-label regimes (exactly our scenario)
- Chen et al., "A Simple Framework for Contrastive Learning" (SimCLR, ICML 2020)
  - NT-Xent loss framework, projection head design

## Architecture

```
                    DXF File (unlabeled)
                           |
                     [Graph Builder]
                           |
                      Raw Graph G
                      /         \
               [Aug 1]           [Aug 2]
                 /                   \
            G_aug1                G_aug2
               |                     |
         [GNN Encoder]         [GNN Encoder]  (shared weights)
               |                     |
            h_1                    h_2         (graph embeddings)
               |                     |
        [Projection Head]    [Projection Head] (shared weights)
               |                     |
            z_1                    z_2         (projected embeddings)
                \                 /
                 [NT-Xent Loss]
```

After pretraining, the **Projection Head is discarded** and the GNN Encoder
is finetuned with a new classification head on the small labeled dataset.

## Graph Augmentation Strategies

| Augmentation        | Default Ratio | Description                                      |
|---------------------|---------------|--------------------------------------------------|
| node_feature_masking | 0.15         | Randomly zero out node features (per-dim)        |
| edge_dropout         | 0.10         | Randomly remove edges                            |
| subgraph_sampling    | 0.80         | BFS-based connected subgraph (80% of nodes)      |
| node_dropout         | 0.10         | Randomly remove nodes and incident edges          |

During pretraining, each view is generated by applying one randomly chosen
augmentation. This follows the GraphCL recommendation of diverse augmentations
to avoid mode collapse.

## GNN Encoder

Reuses the existing `EdgeGraphSageClassifier` / `SimpleGraphClassifier` GNN
layers from `src/ml/train/model_2d.py`. A new `GraphEncoder` class wraps the
GNN layers (without the classification head) for clean reuse.

- 2 EdgeSage/GCN layers
- Global mean pooling
- Output: `[batch_size, hidden_dim]` embeddings

## Projection Head

2-layer MLP: `hidden_dim -> hidden_dim -> 128`

The projection head maps embeddings to a space where the contrastive loss
operates. Per Chen et al., this improves representation quality by allowing
the encoder to retain more task-relevant information.

## NT-Xent Loss

Normalized Temperature-scaled Cross-Entropy (InfoNCE):

```
L = -log( exp(sim(z_i, z_j) / tau) / sum_k exp(sim(z_i, z_k) / tau) )
```

Where:
- `(z_i, z_j)` are positive pairs (two views of the same graph)
- `z_k` iterates over all other examples in the batch (negatives)
- `tau = 0.5` (temperature)
- `sim` = cosine similarity

## Training Configuration

| Parameter     | Default | Notes                                    |
|---------------|---------|------------------------------------------|
| Epochs        | 100     | Contrastive pretraining                  |
| Batch size    | 32      | Larger = more negatives = better loss    |
| Learning rate | 0.001   | With cosine annealing                    |
| Temperature   | 0.5     | NT-Xent temperature                      |
| Hidden dim    | 64      | Matches existing model architecture      |
| Projection dim| 128     | Output of projection head                |

## Finetuning Strategy

Three approaches, in order of data-efficiency:

1. **Frozen encoder** (most data-efficient):
   - Freeze encoder weights, train only the new classifier head
   - Best when labeled data is extremely scarce (<50 samples)

2. **Differential LR** (default, recommended):
   - Encoder: lr=0.0001 (10x lower)
   - Classifier head: lr=0.001
   - Allows encoder to adapt while preserving learned representations

3. **Full finetune** (most flexible):
   - Same LR for all parameters
   - Risks overfitting with very few labeled samples

Early stopping with patience=5 prevents overfitting.

## Integration with Existing Pipeline

The contrastive pretraining pipeline is **additive** -- it does not modify
or replace any existing training code.

New files:
- `src/ml/graph_augmentations.py` - Graph augmentation functions
- `scripts/pretrain_graph2d_contrastive.py` - Contrastive pretraining
- `scripts/finetune_graph2d_from_pretrained.py` - Finetuning from checkpoint

Modified files:
- `src/ml/train/model_2d.py` - Added `GraphEncoder`, `ProjectionHead`,
  `nt_xent_loss`, `get_encoder()`, `load_pretrained()`, `DXF_MAX_NODES`

The existing `run_graph2d_pipeline_local.py` and all model classes continue
to work unchanged.

## Usage Examples

### Contrastive Pretraining (no labels needed)

```bash
# On a directory of unlabeled DXF files
python scripts/pretrain_graph2d_contrastive.py \
    --dxf-dir /path/to/unlabeled_dxf \
    --epochs 100 --batch-size 32 \
    --output models/graph2d_pretrained_contrastive.pth

# Dry-run (CI / testing)
python scripts/pretrain_graph2d_contrastive.py --dry-run --epochs 2
```

### Finetuning (with labeled manifest)

```bash
python scripts/finetune_graph2d_from_pretrained.py \
    --pretrained models/graph2d_pretrained_contrastive.pth \
    --manifest manifest.csv --dxf-dir /path/to/dxf \
    --epochs 50 --patience 5 \
    --output models/graph2d_finetuned.pth

# Dry-run (CI / testing)
python scripts/finetune_graph2d_from_pretrained.py --dry-run --epochs 3
```

### Loading pretrained weights into existing classifier

```python
from src.ml.train.model_2d import EdgeGraphSageClassifier

clf = EdgeGraphSageClassifier(19, 7, 64, num_classes=25)
clf.load_pretrained("models/graph2d_pretrained_contrastive.pth")
# Now finetune clf with your training loop...
```

## Expected Improvement

Based on GraphCL literature (You et al., NeurIPS 2020):
- Molecular property prediction: +2-8% accuracy with contrastive pretraining
- Low-label regime (1-10% labels): up to +15% accuracy improvement
- Our scenario (extreme low-label, 2.3 samples/class): potentially +10-20% absolute improvement

Conservative estimate for our DXF classification:
- From ~27% (supervised only) to ~40-50% with contrastive pretraining + finetuning
- With 25-class taxonomy (B1 consolidation): potentially 50-65%
