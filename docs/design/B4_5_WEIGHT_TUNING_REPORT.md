## 3. 配置更新记录

### 3.1 `src/ml/hybrid_config.py` 变更

```python
# 变更前（B4.4 及之前）
class FilenameClassifierConfig:
    fusion_weight: float = 0.7    # 过度依赖文件名

class Graph2DConfig:
    enabled: bool = False          # ← Graph2D 完全禁用！
    fusion_weight: float = 0.3
    min_confidence: float = 0.5

# 变更后（B4.5）
class FilenameClassifierConfig:
    fusion_weight: float = 0.45   # 降低：Graph2D 更可信

class Graph2DConfig:
    enabled: bool = True           # ← 激活！
    fusion_weight: float = 0.50   # 大幅提升（90.5% 精度）
    min_confidence: float = 0.35  # 降低阈值以接受更多预测
```

### 3.2 权重决策依据

虽然 Grid Search 最优是 fn_w=0.3, g2d_w=0.3（完美文件名假设下的综合 95.2%），但实际系统中文件名置信度不是完美的 1.0。因此采用：

```python
filename_weight = 0.45   # 保留文件名主导地位（实际场景有噪声）
graph2d_weight  = 0.50   # Graph2D 作为强信号（90.5% 精度）
```

**理由**：
- Grid Search 模拟了"完美文件名"（conf=1.0），实际关键词匹配置信度通常 0.6-0.9
- fn_w=0.45, g2d_w=0.50 在搜索中给出 99.0%/90.5%/94.7%，足够优秀
- 保留较高 fn_w 防止嘈杂文件名场景下精度下滑
