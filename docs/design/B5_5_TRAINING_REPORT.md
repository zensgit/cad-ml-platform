## 4. INT8 量化结果（B5.2b）

### 4.1 量化执行

```
Loading checkpoint: models/graph2d_finetuned_24class_v3.pth
Set quantization engine to qnnpack (was: none)

Original  : 1623.3 KB
Quantized :  429.8 KB  (26% of original → 74% smaller)
```

### 4.2 延迟基准测试（合成图 50 节点/100 边）

| 指标 | FP32 | INT8 | 说明 |
|------|------|------|------|
| P50 | 0.6ms | 1.1ms | INT8 在 qnnpack/ARM 上更慢 |
| P95 | 0.6ms | 1.3ms | 量化开销大于小图收益 |
| Speedup | — | 0.5x | 预期在大图/x86 上更优 |

**原因分析**：
- 50 节点的合成图**太小**，量化 pack/unpack 开销占主导
- qnnpack（ARM 优化）在 Apple Silicon 上的 Linear 量化效率不如 fbgemm（x86）
- **生产环境（x86 Linux + 大图）预期表现更好**

### 4.3 qnnpack 引擎修复

**问题**：macOS 默认 PyTorch 量化引擎为 `none`，导致 `RuntimeError: NoQEngine`  
**修复**：在量化前检测引擎，自动切换到 `qnnpack`：

```python
_engine = torch.backends.quantized.engine
if _engine == "none":
    torch.backends.quantized.engine = "qnnpack"
```
