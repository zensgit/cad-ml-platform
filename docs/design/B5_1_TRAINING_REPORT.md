## 2. 技术实现

### 2.1 文字提取器（text_extractor.py）

**支持的 DXF 实体类型**：

| 实体类型 | 描述 | 处理方式 |
|---------|------|---------|
| MTEXT | 多行文字 | `plain_text()` + GBK `\M+5` 解码 |
| TEXT | 单行文字 | `dxf.text` 直接读取 |
| ATTRIB | 块属性实例 | `dxf.text` 直接读取 |
| ATTDEF | 块属性定义 | `dxf.text` 直接读取 |

**双编码支持**：
```python
# 问题：旧版 CAD 工具将中文编码为 \M+5XXXX 格式（GBK 字节序列）
# 解决：正则表达式识别 + bytes.fromhex().decode('gbk')
_M_ESCAPE = re.compile(r"\\M\+5([0-9A-Fa-f]+)")

def _decode_m_escapes(text: str) -> str:
    def repl(m):
        return bytes.fromhex(m.group(1)).decode("gbk", errors="replace")
    return _M_ESCAPE.sub(repl, text)
```

**实测效果**：
```
输入: '\M+5BCFE14\M+5D6E1\M+5B3D0\M+5D7F9'
输出: '轴承座铸件不得有砂眼'  (GBK 解码后的正确中文)
```

### 3.3 关键发现：主类限制

法兰/轴类/箱体 3 个主类占 val set 91%，但关键词命中率仅 5-25%，成为瓶颈：
- **法兰**：DXF 中实际文字常是"NB/T47010"（标准号）而非"法兰"本身
- **箱体**：工程标注"焊接"、"角焊缝"等词与其他焊接件类别共享
- **轴类**：几乎无唯一性关键词（"轴"字在机械图中极为普遍）

## 4. 三路权重搜索结果（实际）

**搜索空间**：fn_w ∈ {0.35-0.50}，g2d_w ∈ {0.35-0.50}，txt_w ∈ {0.10-0.25}，共 64 组合

### 4.1 关键数据

| 场景 | 最优 acc | 推荐配置 |
|------|---------|---------|
| A: 有名+有文字 | **100.0%** | fn=0.45, g2d=0.35, txt=0.10 |
| B: 无名+有文字 | **90.9%** | fn=0.45, g2d=0.35, txt=0.10 |
| C: 纯 Graph2D | 91.0% | 基线不变 |
| 综合 avg | **94.1%** | fn=0.45, g2d=0.35, txt=0.10 |

### 4.2 文字融合实际影响

- **val set 文字命中率**：仅 14.9%（136/914 样本有效命中），因主类 法兰/轴类/箱体 命中率极低
- **场景 B 改善**：90.9% vs 91.0%（基本持平）——文字信号稀疏，对总体精度贡献有限
- **受益类别**：换热器/罐体/过滤器在文字命中时精度 100%，但这些类占比小（<2% 样本）

### 4.3 结论

文字融合对**稀缺专业类别**（换热器、过滤器、罐体）有精准辅助作用，但对主类（法兰/轴类/箱体）贡献有限。推荐以低权重（txt_w=0.10）接入，避免文字噪声降低主类精度。
