# CAD ML Manufacturing Review Manifest Governance Development

Date: 2026-05-13

## Goal

Prevent autogenerated manufacturing evidence suggestions from being counted as
release-reviewed labels. The review manifest validator now distinguishes suggested
labels, approved reviewed labels, and optionally reviewer-owned approved labels.

## Changes

- Updated `scripts/build_manufacturing_review_manifest.py`.
  - Generated manifests now include:
    - `review_status`
    - `reviewer`
    - `reviewed_at`
  - Generated rows still default to `review_status=needs_human_review`.
  - Rows with a `review_status` column only count toward release-reviewed samples when
    status is one of:
    - `accepted`
    - `approved`
    - `confirmed`
    - `release_approved`
    - `reviewed`
  - Rows without a `review_status` column keep backward-compatible behavior.
  - Validation summaries now include:
    - `approved_review_statuses`
    - `approved_review_sample_count`
    - `unapproved_review_sample_count`
    - `require_reviewer_metadata`
    - `reviewer_metadata_missing_sample_count`
  - Added `--require-reviewer-metadata` to require both reviewer identity and review
    timestamp before approved rows count.
- Updated `scripts/ci/build_forward_scorecard_optional.sh`.
  - Supports
    `FORWARD_SCORECARD_MANUFACTURING_REVIEW_MANIFEST_REQUIRE_REVIEWER_METADATA=true`.
  - Passes `--require-reviewer-metadata` to the validator when enabled.
- Updated `.github/workflows/evaluation-report.yml`.
  - Adds repository-variable wiring for
    `FORWARD_SCORECARD_MANUFACTURING_REVIEW_MANIFEST_REQUIRE_REVIEWER_METADATA`.
- Updated `src/core/benchmark/forward_scorecard.py`.
  - Preserves the new governance fields in
    `components.manufacturing_evidence.review_manifest_validation`.
- Updated tests for:
  - generated templates not counting as approved review evidence
  - approved status gating
  - optional reviewer metadata enforcement
  - workflow env wiring
  - scorecard propagation of governance fields

## Review Flow

1. Generate the review manifest from benchmark results.
2. Reviewers inspect `suggested_*` and `actual_manufacturing_evidence`.
3. Reviewers copy approved values into the `reviewed_*` columns.
4. Reviewers set `review_status=approved` or another approved status.
5. If reviewer metadata is required, reviewers fill `reviewer` and `reviewed_at`.
6. CI validates the manifest before the scorecard can rely on the reviewed metrics.

## Release Impact

This removes a subtle release risk: `--prefill-reviewed-from-suggestions` can still
bootstrap fixtures or local dry runs, but those rows remain `needs_human_review` and
do not count toward release-reviewed sample thresholds until an approved review status
is present.
