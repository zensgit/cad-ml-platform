# CAD ML Manufacturing Evidence CI Artifact Development

Date: 2026-05-12

## Goal

Make manufacturing evidence benchmark summaries auditable in CI/release runs. When
the forward scorecard consumes a manufacturing evidence summary from repository
variables or workflow inputs, the workflow now uploads that exact summary as a
separate artifact.

## Changes

- Updated `scripts/ci/build_forward_scorecard_optional.sh`.
  - Tracks the effective manufacturing evidence summary JSON passed into
    `scripts/export_forward_scorecard.py`.
  - Emits `manufacturing_evidence_summary_available`.
  - Emits `manufacturing_evidence_summary_json`.
  - Preserves the previous precedence order:
    1. `BENCHMARK_SCORECARD_MANUFACTURING_EVIDENCE_JSON`
    2. workflow input alias `forward_scorecard_manufacturing_evidence_summary_json`
    3. `FORWARD_SCORECARD_MANUFACTURING_EVIDENCE_SUMMARY_JSON`
- Updated `.github/workflows/evaluation-report.yml`.
  - Adds `Upload manufacturing evidence benchmark summary`.
  - Uploads the exact path reported by the forward scorecard wrapper.
  - Keeps the artifact independent from the rendered forward scorecard so release
    reviewers can inspect the raw evidence coverage source.
- Added workflow regression coverage in
  `tests/unit/test_evaluation_report_workflow_forward_scorecard.py`.
- Extended `tests/unit/test_forward_scorecard_release_gate.py` to assert the wrapper
  emits the manufacturing evidence artifact outputs.

## Artifact Contract

Artifact name:

```text
manufacturing-evidence-benchmark-summary-${{ github.run_number }}
```

Artifact path:

```text
${{ steps.forward_scorecard.outputs.manufacturing_evidence_summary_json }}
```

The upload is gated by:

```text
steps.forward_scorecard.outputs.manufacturing_evidence_summary_available == 'true'
```

## Current Status

The CI/release path can now preserve a real manufacturing evidence benchmark summary
once repository variables point to an existing `summary.json` generated by:

- `scripts/batch_analyze_dxf_local.py`
- `scripts/eval_hybrid_dxf_manifest.py`

Remaining work is now about evidence quality, not artifact plumbing: add reviewed
labels and source correctness metrics for DFM, process, cost, and manufacturing
decision evidence.
